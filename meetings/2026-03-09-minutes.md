## Signature based SRI

Seems useful https://wicg.github.io/signature-based-sri/. Shipping in Chromiums 141. We could document this.

Webkit: https://github.com/WebKit/standards-positions/issues/434
Firefox: https://github.com/mozilla/standards-positions/issues/1139
TAG review: https://github.com/w3ctag/design-reviews/issues/1041

Florian: survey also showed that SRI is annoying to implement.. this seems to address that. Potential cross browser support... 

Will: SRI is brittle - any time the hash changes it breaks... has to be recalculated... comoplicated. So instead of a hash you use a signature - in the markup you have a key instead of the hash. Include signatures as headers... You can generate different signatures... the browser gets the signature from the header... 

Dan: new header?

Will: yes, a new header.

Dan: sounds also more complicated... CDNs would need to support as well...

Will: looks early to me to document on MDN...
